# ft_transcendence: Event Recommendation Engine

A content-based recommendation system designed to serve personalized Ticketmaster events to users based on their interaction history and explicit preferences.

## Architecture

* **Backend:** Python (scikit-learn)
* **Frontend:** JavaScript
* **Data Source:** Ticketmaster API

## Core Algorithm

The recommendation engine mathematically matches users to events using the following pipeline:

1. **Event Vectorization:** Event metadata (types, sub-types, categories) is converted into a mathematical matrix using feature extraction (e.g., `CountVectorizer`).
2. **User Profiling:** A single Weighted User Profile Vector ($V_u$) is generated by combining a user's explicit preferences with their behavioral history.
* Interactions are assigned specific weights: Attended (highest), Liked/Saved (high), Viewed (scaled by duration), and Disliked/Dismissed (negative).


3. **Similarity Scoring:** Cosine Similarity is calculated between the User Profile Vector and the new Event Vectors ($V_{new}$) to determine the most relevant matches:

$$\text{Score} = \frac{V_u \cdot V_{new}}{\|V_u\| \|V_{new}\|}$$


4. **Delivery:** The backend sorts the events by score and serializes the top 20 into a lightweight JSON payload (containing only ID, Title, Thumbnail, Date, and Venue) for the frontend to render.

## Data Structures

User profiles are maintained in a JSON structure, tracking:

* **Preferences:** Explicitly chosen segments, genres, and price ranges.
* **History:** Arrays mapping interactions like `viewedEventIds`, `likedEventIds`, `attendedEventIds`, and `dislikedEventIds` to continuously update the profile vector.
